# KimiCode 课件 · Vibe Coding 从入门到上线

> 面向零基础（非技术背景）大学生的七节课实战课程
> 核心理念：**先描述想法，让 AI 帮你实现，不必先学语法**
> 每节课都做出一个能展示、能上手的真实成品

---

## 课程总览

| 节次 | 主题 | 当节产出 | 新学到的能力 | 需课前准备 |
|------|------|----------|--------------|------------|
| 第一节 | 先做一个很酷的东西 | 3D 可交互月球网页 | 拆解并复用一个有效 prompt | — |
| 第二节 | 造一个简易版 Manus | 会自己规划、调用多个 skill 的 agent | agent 概念、skill、orchestration prompt、MCP | IDE + CLI 配置 |
| 第三节 | 把 Manus 做成真正的项目 | 接真实 API 的 Next.js 网页（本地） | Git、Node、Next、.env、调 API | GitHub 账号 |
| 第四节 | 定时舆情分析网页 | 能登录、定时抓取、分析的看板 | 官方 API、数据库、登录验证、定时任务 | Reddit/GitHub API key |
| 第五节 | 做一个 Chrome 扩展 | 装进自己浏览器的实用小工具 | 扩展结构、content script、popup | — |
| 第六节 | 做一个微信小程序 | 能跑通的小程序 | 小程序结构、微信开发者工具、审核流程 | 小程序账号（实名慢，务必提前） |
| 第七节 | 封装与部署 | 把前面每个项目都送上线 | 部署、环境变量上线、云端定时、商店/审核 | 各平台账号 |

**贯穿全程的一条原则**：命令记不住没关系，遇到报错就截图丢给 AI，让它教你修。这本身就是 vibe coding 的工作方式。

## 第一节课：先做一个很酷的东西（The Wow Moment）

**目标**：第一节课就让学生用一句话造出一个看起来很厉害的 3D 交互月球，打掉「编程很难」的心理门槛。

**时长**：约 60 分钟

### 流程

**1. 开场惊叹（5 min）**
直接展示做好的月球——可以缩放、旋转看背面、背景有星星。问学生「你们猜这要做多久」，然后揭晓：一句话。

**2. 认识工具（5 min）**
打开 KimiCode，介绍最核心的两个区域：
- **对话框**——你说话的地方
- **预览窗口**——结果实时出现的地方

不讲任何术语。

**3. 跟着做（15 min）**
把这条已验证有效的 prompt 复制进去运行：

```
Create a HTML webpage based on the following cues. The page should be
a 3D interactive world titled Moon. It features a Moon in space, with
realistic SVG craters and meshes. Users can zoom in and out, and rotate
the moon (to see the dark side of it). The space should have little stars.
```

让每个学生都亲眼看到月球被生成、亲手转一转。

**4. 拆解这条 prompt（10 min）——本节最重要的知识点**

逐句讲为什么它有效，提炼出一个可复用的结构：

| 要素 | 作用 | 本例 |
|------|------|------|
| 是什么 | 标题 / 类型 | a 3D interactive world titled Moon |
| 主体 | 核心对象 | a Moon in space |
| 细节 | 让它真实 | realistic SVG craters and meshes |
| 交互 | 用户能做什么 | zoom in/out, rotate to see the dark side |
| 氛围 | 锦上添花 | little stars |

**结论**：描述得越具体，AI 给的结果越接近你想要的。

**5. 自己改（15 min）**
让学生在这条 prompt 上动手改，做出自己的版本。引导方向：
- 换成火星、土星（带光环）
- 改月球颜色、加一行说明文字
- 让星星会闪烁
- 加一个能点击的标记点

鼓励大胆试错，不满意就继续对 AI 说「再改一下……」。

**6. 分享收尾（5 min）**
挑几个学生作品展示，总结心法：**把想法描述清楚，就是编程。**

### 课后作业
用今天学的 prompt 结构，做一个属于自己的 3D 场景（地球、太阳系、虚构星球），或给月球加一个新功能。

## 第二节课：造一个你自己的简易版 Manus

**目标**：从「我会让 AI 做一件事」跨越到「我造了一个能自己规划、调动多种能力、把复杂任务自动跑完的 agent」。

**核心立意**：Manus 这类 agent 的厉害之处，不是会某一个绝活，而是**会规划 + 会调用一堆不同的 skill**把一件复杂的事从头做到尾。
> **agent = 会规划的大脑 + 一堆 skill**

**时长**：约 75 分钟

### 流程

**1. 开场惊叹（5 min）**
放一段简易版 Manus 的演示：给它一句话总任务（例：「帮我研究『电动车 vs 燃油车』，产出一份报告和一套幻灯片」），让学生看着它自己拆步骤、逐个调用不同 skill、最后吐出成品。
> 「上节课我们一句话造了个月球，这节课我们造一个会替你打工的助理。」

**2. 什么是 agent（10 min）**
只用一个比喻，不堆术语：
- 上节课的 AI 像一个「一次性工人」——你说一句，它做一件事
- agent 像一个「项目负责人」——你给它大目标，它自己想「分几步、每步用什么本事」，逐步完成，还能根据结果调整
- 它的「本事」就是上节课的 **skill**；agent 强不强，取决于手里有多少趁手的 skill

**3. 环境就位（10 min，建议设为课前预习）**
装好 IDE（VS Code）、装好并登录 KimiCode CLI、跑通第一条命令。
> ⚠️ 强烈建议把配置做成课前预习视频/手册，课上只花 10 分钟集体排查，把课堂火力留给造 agent。

**4. 动手造（核心环节，约 40 min）**

**第一步——定一个总任务（5 min）**
选一个「一步做不完、必须分好几步」的目标：
- 研究型：研究某主题 → 报告 + 幻灯片
- 策划型：规划三天旅行 → 行程表 + 预算表 + 打包清单
- 产品型：一句创意 → 落地页 + 宣传文案

任务要「够大」，大到必须拆步骤、用到不止一种能力。

**第二步——准备这个 agent 的「本事」（多个 skill）（15 min）**
以研究型为例，至少需要：

| skill | 负责什么 |
|-------|----------|
| 调研 skill | 怎么找资料、判断可信、整理要点 |
| 写报告 skill | 结构、语气、格式规范 |
| 做幻灯片 skill | 每页怎么排、风格统一 |
| 统一风格 skill | 颜色、字体、品牌调性 |

先用现成 skill（如 superpower 系列）做**开 vs 不开**的对比演示，让学生直观感受 skill 的价值；然后让每人**至少自己写一个 skill**（最简单：「我的风格偏好 skill」或「我的报告结构 skill」），亲手给 agent 加一项本事。

**第三步——写「大脑」：让它学会自己规划和调度（15 min）**
这是和第一节最不同的一步。带学生写 orchestration prompt（agent 主指令），教 AI 三件事：
1. **先规划**：拿到总任务，先列步骤，不急着动手
2. **再按需调用**：每一步该用哪个 skill 就调哪个
3. **串起来**：上一步产出当作下一步输入，最后汇总成完整交付物

可填空骨架模板：

```
你是一个项目负责人。收到任务后，先把它拆解成步骤，并说明每步用哪个能力；
然后逐步执行，每一步调用对应的 skill；
全部执行完后，汇总所有产出成为完整的交付物。

任务是：______
```

让学生填进自己的总任务跑起来，看它自己「先想后做」——这就是本节的高光时刻。

**第四步（进阶/选做）——接 MCP，让它够得到真实世界**
MCP 让 agent 从「凭脑子工作」升级到能读真实文件、查真实数据、把产出存到网盘/日历。现场演示接一个最直观的 MCP（文件系统或网盘），让 agent 把报告和幻灯片真正存成文件。

**5. 自己扩展（10 min）**
给自己的 agent 再加一项本事：多写一个 skill、多接一个 MCP，或给总任务加一个新步骤（如「自动写一封推荐这份报告的邮件」）。

**6. 收尾（5 min）**
> 「上节课你学会把一个想法说清楚；这节课你学会把一个复杂目标交给一个会规划、会调动多种本事的 agent。你今天造的，就是一个属于你自己的迷你版 Manus。」

### 课后作业
给简易 Manus 至少配齐 **3 个 skill**、完成一个完整的多步任务，并交一份「运行记录」：它把任务拆成哪几步、每步用了什么 skill、踩了什么坑、怎么靠 AI 修好的。

### 节奏提醒
信息量大，一节课很可能塞不下。建议：IDE/CLI 配置全部前置到课前；「自己写 skill」压到只写一个、其余用现成的；火力集中在「规划 + 调度」。若仍紧张，把「接 MCP」整块挪到第三节。

## 第三节课：把简易 Manus 做成一个真正的项目

**目标**：把 agent 从 KimiCode 环境「搬」成一个真正的 Next.js 网页项目，第一次接触工程化四件套——版本管理、运行时、框架、密钥管理，并接真实 API。**本节只在本地跑通，不部署**（部署留到第七节）。

**时长**：约 75 分钟

### 流程

**1. 为什么要「搬家」（5 min）**
上节课的 agent 困在对话里——别人用不了、改了容易丢、接不了你自己的密钥。做成真正的网页项目，这些都解锁。

**2. Git：给项目上「存档点」（15 min）**
用游戏存档类比：commit = 存档，出错能读档回去，多人协作还能合并进度。只教核心几步：
- 初始化
- add
- commit
- 看历史
- 回退

建 GitHub 账号，把项目推上去。

**3. Node.js + Next.js：认识工作台（15 min）**
用最朴素的话讲清：
- **Node** = 让 JavaScript 在你电脑上跑起来的引擎
- **Next** = 帮你快速搭网页的框架，前端页面和后端逻辑都能写

让 AI 帮忙用 Next 起一个最小项目，跑起来看到本地页面，体会「改文件、网页自动刷新」。

**4. 环境变量 .env：藏好你的钥匙（10 min）——本节最重要的工程意识**
- API key 是你的钥匙，**绝对不能直接写在代码里、更不能推到 GitHub 上**（别人捡到能拿你的账号花钱）
- 把密钥放进 `.env` 文件
- 把 `.env` 加进 `.gitignore`，让它永远不会被上传

> 🔑 这个习惯越早建立越好。

**5. 接真实 API，让 agent 长出「真本事」（25 min）——核心动手环节**
让简易 Manus 接上真实大模型 API（用 `.env` 里的 key），从「在对话框里跑」变成「在自己的网页里、用自己的密钥跑」。做出一个能用的界面：
- 一个输入框接收总任务
- 后端调用 API 做规划和分步执行
- 结果展示在页面上

能力强的把第二节的多 skill 调度逻辑也搬进来。

**6. 收尾（5 min）**
> 「今天你的 agent 从一个聊天记录，变成了一个真正属于你自己的网页应用——有版本、有引擎、有框架，钥匙也藏好了。」

### 课后作业
- 把项目完整推到自己的 GitHub
- 在本地跑通接了真实 API 的简易 Manus
- **专门检查 `.env` 没有被上传**（安全红线）

## 第四节课：做一个会自己定时跑的舆情分析网页

**目标**：从「用户点一下才动」进阶到「自己定时抓数据、存起来、做分析」。第一次接触数据获取、用户系统、数据持久化、定时任务。建议主攻 **Reddit 或 GitHub**——都有官方 API，合规、稳定、好上手。

**时长**：约 80 分钟

### 流程

**1. 看成品（5 min）**
展示一个舆情面板：定时抓取某话题在 Reddit/GitHub 上的讨论，自动分析情绪和热点，登录后才能看到自己的看板。

**2. 数据从哪来：优先用官方 API，而不是硬爬（15 min）**
建立正确观念：**能用官方 API 就别硬爬。**
- Reddit / GitHub 都提供官方 API：合规、稳定、附带结构化数据
- 教学生申请 API key（存进 `.env`），用 API 拉一个话题的帖子/issue 回来
- 尊重使用条款和频率限制，别把人家服务器打挂——这是技术规范也是基本礼貌
- 确实只能从网页拿的数据，再简单演示基础抓取，同样守规矩

**3. 数据库：把抓回来的东西存下来（15 min）**
为什么要数据库——抓回来的数据不能只放内存里，关掉就没了。用上手简单的方案（如 Supabase 这类带现成后端的）：
- 建一张表
- 把抓到的内容存进去
- 再读出来展示

体会「写入—读取」这个最基本的循环。

**4. 登录验证：这是谁的看板（15 min）**
为什么要登录——不同用户看不同数据，有些内容不该公开。用现成认证方案（Supabase Auth / Clerk 这类）加注册登录，做到「登录后才能看自己的看板」。先会用、理解它解决了什么问题即可。

**5. 分析 + 定时跑（15 min）**
- 用大模型 API 对抓回来的讨论做情绪/热点分析（复用第三节经验）
- 讲「定时任务」：让程序每隔一段时间自己跑一次抓取和分析
- 本地先用最简单的定时方式演示效果，**真正的云端定时留到第七节落地**

**6. 收尾（5 min）**
> 「今天你的应用第一次有了记忆（数据库）、有了门禁（登录）、还会自己干活（定时）。这已经是一个有模有样的真实产品了。」

### 课后作业
做出一个能登录、能抓取某话题、能展示分析结果的舆情面板；写一段说明：数据来自哪个官方 API、你怎么避免触犯它的频率限制。

## 第五节课：做一个 Chrome 扩展

**目标**：换一种产品形态——不是一个网页，而是一个「长在别人网页上」的小工具。理解扩展怎么运行，做出一个能装进自己浏览器、立刻能用的东西。

**时长**：约 70 分钟

### 流程

**1. 看成品（5 min）**
展示一两个实用小扩展：
- 一键总结当前网页
- 把选中文字翻译/改写
- 把当前页面信息存进第四节的数据库

**2. 扩展是怎么回事（15 min）**
用最直白的方式讲清结构：

| 组成 | 作用 |
|------|------|
| manifest | 配置文件，说明这个扩展是谁、要什么权限 |
| content script | 运行在你正在浏览的网页上的代码 |
| popup | 工具栏点开的弹出小窗（可选） |

讲清「哪段代码在哪里跑」就够了。

**3. 跑通第一个扩展（20 min）**
让 AI 帮忙生成一个最小扩展，在 Chrome 用「开发者模式」加载，亲眼看到它出现在工具栏、点一下有反应。
> 「我做的东西装进了我自己的浏览器」——这个时刻很有冲击力。

**4. 做一个真正有用的（25 min）——核心动手环节**
做一个调用大模型 API 的小扩展，比如总结或改写当前网页选中的内容（复用前几节接 API 经验）。能力强的让它和第四节项目联动——把网页内容一键存进自己的数据库。

**5. 收尾（5 min）**
> 「同一身本事（接 API、连数据），可以做成网页，也可以做成扩展。产品形态不同，内核是相通的。」

### 课后作业
做出一个自己真的会用的 Chrome 扩展并装进浏览器，录一段使用演示。

## 第六节课：做一个微信小程序

**目标**：再换一种形态——从浏览器走到手机、走到微信生态。面向中国大学生特别实用，小程序是国内分发产品最贴近用户的方式之一。

**时长**：约 75 分钟

### 流程

**1. 看成品（5 min）**
展示一个简单但完整的小程序，最好和前面项目呼应（如把舆情面板或某个工具做成小程序版）。

**2. 小程序是怎么回事（15 min）**
讲清它和普通网页的区别：
- 跑在微信里、用微信的账号体系
- 有自己的一套页面写法和组件
- 发布前要走微信审核

**3. 环境与账号（15 min，建议部分课前准备）**
- 装好微信开发者工具
- 注册小程序账号
> ⚠️ 实名和资质流程很慢，**强烈建议课前就让学生提前申请**，否则课上会卡住一大片人。
- 课上集中演示工具界面、跑通官方 hello world

**4. 做一个真正的小程序（35 min）——核心动手环节**
让 AI 帮忙搭一个有实际功能的小程序：
- 页面
- 用户能操作的交互
- 调用后端 API 拿数据

可以把前面某个项目（舆情面板/agent/工具）做成小程序版，复用已有的后端和 API。在开发者工具的模拟器里看它跑起来，有条件用真机预览，体验「用微信扫一下，在自己手机上打开了我做的东西」。

**5. 收尾（5 min）**
> 「网页、扩展、小程序——你已经把同一套能力，送到了三种完全不同的地方：浏览器、别人的网页、和每个人的微信里。」

### 课后作业
做出一个能在开发者工具里完整跑通的小程序；有条件用真机预览并录屏。

## 第七节课：封装与部署——把前面每个项目都送上线

**目标**：全课程收口。前面每节都做了能在本地跑的东西，但「只有我电脑上能跑」不算完成。统一教部署，让每个项目都变成「任何人点开链接/扫码就能用」的真实产品。

**时长**：约 90 分钟

### 流程

**1. 为什么部署是「最后一公里」（5 min）**
做出来只是一半，让别人用得上才算完整。今天把前六节的成果一个个真正送上线。

**2. 通用观念：部署前的检查清单（15 min）——本节最值钱的工程素养**
建立一套可复用的「上线前体检」：
- 密钥有没有藏好（`.env` 不能上线泄露，云端要用平台的环境变量配置）
- 有没有把不该公开的东西公开
- 免费额度够不够用
- 出错了怎么看日志

**3. 逐个项目部署（60 min）**

**网页类**（第三节简易 Manus、第四节舆情面板）
- 部署到 Vercel 这类平台，代码连上 GitHub 实现自动上线
- `.env` 里的密钥搬到平台的环境变量
- 第四节的定时任务在这一步落地成**云端定时**，实现无人值守自动跑

**数据库**
- 确认线上连的是正式数据库，权限设置正确

**Chrome 扩展**（第五节）
- 分发两条路：打包成文件直接发给别人装 / 走 Chrome 应用商店审核上架（商店审核需时间和开发者账号）

**微信小程序**（第六节）
- 从开发者工具提交代码 → 走微信审核 → 发布上线，强调审核需时间、要符合平台规范

**4. 结课与回望（10 min）**
回看整条路：从一句话造月球 → 造会规划的 agent → 搬成真项目、连真实数据 → 做成扩展和小程序 → 全部送上线。
> 「七节课前，你以为编程很难；现在，你已经把想法变成了能在网页、浏览器、手机上真正运行、别人也能用的产品。」

### 课后作业（结课项目）
选一个自己最满意的项目，完整部署上线，拿到一个能分享的链接或二维码，做一次结课展示。

## 给授课老师的统一提醒

**1. 越往后越要把环境配置/账号申请前置到课前。**
第六节的微信小程序账号、第七节的各平台账号，审核和实名流程都慢，课上现申请必卡。建议每节课前发一份预习清单，把慢流程提前启动。

**2. 部署涉及真金白银的额度。**
云平台、数据库、API 都有免费额度但也都可能超额扣费。第七节务必讲清怎么看用量、怎么设上限。**尤其 API key 泄露导致被盗刷，是最常见也最伤的坑，值得反复强调。**

**3. KimiCode 自身能力边界需自行确认。**
本课纲按通用全栈 + 多形态开发的逻辑搭骨架，但 KimiCode 在每一步具体能帮到什么程度（能否直接帮学生跑 git、起 Next 项目、接 Reddit API、生成小程序代码、对接 Vercel 部署），需要落地前亲自完整跑通一遍，把通用步骤替换成 KimiCode 里准确的操作和截图。

**4. 每节课的 demo 务必课前录屏备份。**
3D 月球、agent 自主规划这类演示对环境和网络敏感，现场掉链子时能直接放录屏，保住每节课的「高光时刻」。
