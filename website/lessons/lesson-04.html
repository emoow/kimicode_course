Lesson 04 · Dashboard

定时舆情分析网页

从“用户点一下才动”进阶到“系统自己定时抓数据、存数据、做分析”。

时长约 80 分钟
产出能登录、定时抓取、分析的看板
准备Reddit 或 GitHub API key

这一节还没有解锁

请先完成前一节，再进入本节。

学习目标

用合规方式获取和分析真实数据。

  • 优先使用官方 API，而不是硬爬。
  • 把抓到的数据写入数据库。
  • 给看板加登录验证和定时任务。
课堂流程

从数据源到用户看板

  1. 申请并保存 API key。
  2. 拉取 Reddit 或 GitHub 话题数据。
  3. 设计数据库表并完成写入读取。
  4. 加入登录后可见的看板页面。
课后作业

提交舆情面板

提交可运行看板截图、数据表结构说明和一次定时抓取记录。
